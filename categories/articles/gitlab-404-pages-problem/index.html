The problem

Custom 404 pages stopped working a year ago for *.gitlab.io domains. A 404 page is a page which you get when a URL can’t be found on a website.

The Gitlab issue .

Why this is a major bug

Frameworks and libraries are more difficult to use

Many libraries like React show incredibly useful error messages on the 404 page when stuff stops working.

It can break the SEO for websites

SEO means how high you rank in search engines. Backlinks (links from other websites to your website) are very important for SEO.

Those backlinks can link to a completely broken page if a website decides to change some URLs and does not have a working 404 page.

The website should show a 404 page instead. Many of my backlinks are going to be invalid now, which is not great.

Why this bothers me

Well, it is going to affect my SEO. The issue was created last year. I think that such an issue is way too important to ignore. It isn’t a minor thing.

Solutions

Buy a domain

Another solution is buy yourself a domain and use it. It does work if you use another domain.

Move to another platform

People are going to hate this, but moving to another platform is a solution. Github does not have this problem.

I would suggest that you host your website there instead until Gitlab fixes this. They will probably lose some users because of this bug, but I overall like Gitlab.

Sure, Gitlab uses the Google servers to host your site on, but it still feels more open than Github (which is owned by Microsoft) and it has wonderful pipelines.

Gitlab has pipelines, which is really useful. It is a reason to move for many people.

What will this website do?

I might move to Github pages if this stays broken. I don’t want to wait much longer for 404 page support.

Codeberg had the same problem and it only took them a few hours to fix it, not even a day. I will probably move to codeberg pages.

I migrated my site to codeberg.

Moving also means that I would lose much SEO, but it might also mean that it might not happen again.

Why don’t you fix it yourself?

Well, I have other stuff to do and I believe that Gitlab has a pretty big codebase. It would require much time.
